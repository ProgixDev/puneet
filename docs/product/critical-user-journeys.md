# Critical user journeys (CUJs)

The flows that must never break, inferred from the central routes. **Every row is `[inferred — confirm]`** until the product owner confirms it. Test coverage is **"none — untested"** across the board: the project has no test runner (see [../quality/debt-map.md](../quality/debt-map.md)).

**Rule going forward:** a change that touches a CUJ requires verifying that journey (run the app, walk the steps) before claiming done. A new user-visible feature must extend an existing CUJ or register a new one here.

## When a test runner arrives

Backfill coverage starting at the top of this table (CUJ-1 → CUJ-5 are the operational spine). Until then, each CUJ's "test" is a documented manual walk-through recorded in the verifying change. Adding the runner is its own change + ADR (see [../architecture/decisions/](../architecture/decisions/)).
